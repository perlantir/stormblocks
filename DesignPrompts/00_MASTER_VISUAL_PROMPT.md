# Master Visual Prompt

Create high-fidelity production-quality mobile game visuals for a free portrait iPhone puzzle game called “Storm Blocks.”

The game is a mainstream premium casual 3D block puzzle with a unique rescue-storm twist. The player drags colorful block pieces onto an 8x8 grid, clears rows and columns, rescues tiny survivors, and pushes back a living storm before it reaches a warm central camp.

Visual style:
Premium Casual Toy-Puzzle Storm Rescue. Bright, polished, rounded, juicy, colorful, highly readable, and studio-quality. Use soft 3D toy-like blocks, rounded UI panels, satisfying particle effects, warm camp lighting, cute tiny survivors, and dramatic blue-purple storm clouds.

Mood:
Easy to understand, addictive, cute, satisfying, hopeful, and exciting. The storm should feel threatening but not scary. The camp should feel warm and worth protecting.

Requirements:
- Portrait iPhone screen, 9:19.5 aspect ratio.
- 8x8 puzzle grid centered on screen.
- Warm central camp/shelter zone.
- Storm tiles creeping inward from the outer edges.
- Three draggable block pieces in a bottom tray.
- Top HUD with score, survivors rescued, and best/daily score.
- Big readable touch targets.
- Rounded buttons and soft shadows.
- Clear contrast between empty cells, placed blocks, storm cells, survivor cells, and camp cells.
- Strong visual focus on automatic Storm Pushback.

Avoid:
Gritty apocalypse, horror, gore, dark realism, muddy colors, cluttered UI, tiny unreadable text, generic Unity UI, flat programmer art, overly serious survival strategy look, casino visuals, gacha visuals, loot boxes, battle pass, shop-focused UI, copyrighted characters, existing game logos, messy grid, unclear blocks, too many currencies, low-resolution icons, blurry UI, hard-to-read board, clone of Block Blast.
