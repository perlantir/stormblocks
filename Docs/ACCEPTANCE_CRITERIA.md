# Acceptance Criteria

## Global

A gate is not complete unless:

- The project compiles.
- Relevant tests pass or failures are documented and intentionally deferred.
- The main gameplay remains playable.
- The implementation log is updated.
- The final product direction remains full-game, not prototype.
- No pay-to-win or copyrighted IP is introduced.

## Design quality

- No default Unity-looking UI in release candidate.
- 8x8 board is readable on iPhone portrait.
- Buttons are large enough for touch.
- Storm VFX never hides playable cells for more than brief celebratory moments.
- The camp is emotionally central but does not confuse gameplay rules.
- Core HUD is minimal and clear.

## Gameplay quality

- First 10 seconds teach drag/place/clear/rescue/pushback visually.
- Storm Pushback is obvious and satisfying.
- Losing leads to immediate retry.
- Daily Storm feels distinct and habit-forming.
- Endless Storm has clear difficulty escalation.
- Storm Trail teaches mechanics and provides long-form progression.

## Full game acceptance

The release candidate must include:

- Endless Storm.
- Daily Storm.
- Storm Trail.
- Tempest Trials.
- Practice/Chill Mode.
- Profile/save.
- Cosmetics.
- Achievements.
- Settings/accessibility.
- Audio/haptics.
- 3D board/pieces/storm/camp/survivor presentation.
- Game Center-ready service layer.
- iOS build documentation.
